# AWS CONTINOUS COMPLIANCE AND CHANGE MANAGEMENT

## Objective

The primary objective of this project is to implement AWS Continuous Compliance and Change Management. This is achieved by creating an automated system that records, evaluates, and alerts on the configuration status of AWS resources to ensure they remain compliant with predefined security rules

### Skills Learned

- Cloud Governance & Compliance
- Security Automation & Alerting
- Storage Security (s3)
- Troubleshooting & Testing

### Tools Used

- Amazon Simple Notification Service (SNS)
- Amazon S3
- AWS Identity and Access Management (IAM)
- AWS CloudTrail
- AWS Config
- Management & Monitoring Tools
- Gmail

### Process

- Set up AWS Config (The Recorder)
- Set up Amazon SNS (The Messenger)
- Add Compliance Rule and Link SNS
- The Break and Fix Cycle (The Test)

### Outcome 

The outcome of the project was the successful establishment of a fully functional automated security and compliance monitoring loop within an AWS environment.
